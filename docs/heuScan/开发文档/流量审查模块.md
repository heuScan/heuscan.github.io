# Passive Traffic Audit Agent 设计与实现文档

## 1. 模块任务描述

### 1.1 模块定位
- **模块名称**：Passive Traffic Audit Agent (被动流量审查模块)
- **所属层级**：聚合后端 - 被动风险审查模块
- **核心职责**：对 HTTP 流量样本进行被动式分析，识别潜在的安全风险线索（如 SQL 注入痕迹、XSS 痕迹、敏感文件泄露、弱凭证传输等），并标注高风险行为。

### 1.2 输入与输出

| 项目 | 描述 | 格式/结构 |
| :--- | :--- | :--- |
| **主要输入** | HTTP 流量样本（包含 Request/Response 对） | `FlowSample` JSON Object |
| **主要输出** | 风险线索列表 | `List[SecurityFinding]` |
| **运行范式** | REACT (推理-行动) + REFLECT (反思-修正) | 内部 Agent 逻辑循环 |

### 1.3 在 Runtime 中的角色
根据 `5.1 Runtime 总体结构` 规定，本模块定位为：
- **角色**：`Specialist Agent` (能力域执行单元)
- **触发者**：仅由 `Orchestrator Agent` 触发。
- **协作方式**：不直接调用 HTTP，不共享内存，仅通过 `AgentTask` 和 `ObservationEvent` 与 `Orchestrator` 交互。

## 3. 代码处理流程

### 3.1 主流程时序图

```
Orchestrator -> Agent Executor: execute(task)
Agent Executor -> PassiveTrafficAuditAgent: run(context)
PassiveTrafficAuditAgent -> REACT Loop: while not done
    REACT Loop -> Tool: call_detector(request_part)
    Tool -> REACT Loop: observation
    REACT Loop -> REFLECT: check_confidence(observation)
    REFLECT -> REACT Loop: adjust_focus / req_more_context
PassiveTrafficAuditAgent -> Agent Executor: yield ReviewTicket
Agent Executor -> Result Aggregator: persist(ticket)
```

### 3.2 详细处理逻辑步骤

#### 步骤 1：任务初始化与上下文解析
- **输入**：`AgentContext` 对象，包含 `FlowSample` 数据。
- **动作**：解析 HTTP 请求头、请求体、响应头、响应体。
- **约束**：仅读取数据，不修改原始上下文。

#### 步骤 2：REACT 推理阶段
本模块采用轻量级 ReAct 范式，不依赖大模型 API，而是基于规则引擎的 Action-Observation 循环。

| 阶段 | 动作 | 代码模块 |
| :--- | :--- | :--- |
| **Thought** | 分析 URL 结构和参数特征 | `HeuristicAnalyzer.think()` |
| **Action** | 调用特定检测工具（如 `SQLiDetector`） | `Toolbox.run(action_name)` |
| **Observation** | 获取工具返回的匹配结果和置信度 | `ObservationEvent` |

#### 步骤 3：REFLECT 反思阶段
- **目标**：降低误报，确认风险等级。
- **逻辑**：
    1. 检查 Observation 中的置信度得分。
    2. 若置信度 > 0.8，直接标记为 **HIGH**。
    3. 若置信度在 0.5~0.8 之间，**REFLECT** 检查上下文（如响应是否返回了数据库错误详情）。
    4. 若置信度 < 0.5，**DROP** 线索。

#### 步骤 4：生成审计票据
- **输出对象**：`ReviewTicket`
- **内容**：包含 `finding_id`，`signature_name`，`matched_payload`，`risk_level`。

## 4. 关键代码分析

### 4.1 Agent 核心定义
严格遵守 Runtime 规范，Agent 不直接暴露 HTTP 接口，只实现标准执行方法。

```python
# agent/specialist/passive_traffic_audit.py

from runtime.base import SpecialistAgent, AgentContext, ObservationEvent, ReviewTicket
from .react_engine import TrafficReactEngine

class PassiveTrafficAuditAgent(SpecialistAgent):
    """
    被动流量审计专家
    职责：流量线索识别与高风险标注
    """
    agent_type = "PassiveTrafficAudit"
    
    def __init__(self):
        self.react_engine = TrafficReactEngine()
        # 初始化检测工具集
        self.tools = {
            "sql_injection": SQLiPatternDetector(),
            "xss": XSSPatternDetector(),
            "path_traversal": PathTraversalDetector()
        }

    async def execute(self, context: AgentContext):
        """
        核心执行入口，由 Agent Executor 调用
        """
        flow_sample = context.input_data.get("flow_sample")
        
        findings = []
        max_iterations = 5  # 防止 REACT 无限循环
        
        # 1. REACT + REFLECT 主循环
        for iteration in range(max_iterations):
            # Thought: 决定下一步检查哪个部分
            focus_area = self.react_engine.think(flow_sample, findings)
            if focus_area is None:
                break  # 无更多可疑区域
                
            # Action: 调用工具检测
            tool = self.tools.get(focus_area.tool_name)
            if not tool:
                continue
                
            observation_data = tool.detect(focus_area.payload, flow_sample.response_body)
            
            # Observation: 生成事件
            event = ObservationEvent(
                agent_name=self.agent_type,
                data=observation_data,
                confidence=observation_data.get("confidence", 0.0)
            )
            
            # Reflect: 反思阶段 - 评估观察结果并决定是否生成线索
            finding = self._reflect_and_decide(event, flow_sample)
            if finding:
                findings.append(finding)
                
            # 上下文更新 (防止上下文膨胀)
            context.update_variable("last_observation", event.data)

        # 2. 高风险标注与聚合
        risk_level = self._calculate_overall_risk(findings)
        
        # 3. 输出审计票据 (符合规范：Agent 不直接写库，返回给 Executor)
        return ReviewTicket(
            task_id=context.task_id,
            agent_type=self.agent_type,
            status="COMPLETED",
            findings=findings,
            risk_level=risk_level,
            audit_trail=context.get_audit_log()
        )

    def _reflect_and_decide(self, event: ObservationEvent, flow_sample):
        """
        REFLECT 核心逻辑：根据置信度和上下文反射判断
        """
        if event.confidence > 0.85:
            return SecurityFinding(risk="HIGH", detail=event.data)
        elif event.confidence > 0.5:
            # 反射验证：检查响应体中是否包含典型的错误堆栈
            if self._has_error_disclosure(flow_sample.response_body, event.data):
                return SecurityFinding(risk="MEDIUM", detail=event.data)
        return None
```

### 4.4 检测工具实现示例 (SQLi)

```python
# tools/sqli_detector.py
import re

class SQLiPatternDetector:
    """
    基于正则和上下文语法的 SQL 注入特征检测
    """
    SQL_KEYWORDS = r'\b(select|union|insert|update|delete|drop|exec|script|sleep|waitfor|delay)\b'
    SQL_EVASION = r'(\'|"|--|#|/\*|\\)'
    SQL_ERRORS = r'(SQL syntax|MySQLSyntaxErrorException|ORA-[0-9]{5}|PostgreSQL|unclosed quotation mark)'

    def detect(self, payload: str, response_body: str) -> dict:
        confidence = 0.0
        matches = []
        
        # 1. 检查 payload 中是否有 SQL 保留字和逻辑运算符
        if re.search(self.SQL_KEYWORDS, payload, re.IGNORECASE):
            confidence += 0.4
            matches.append("SQL Keyword Match")
            
        # 2. 检查是否包含注释符、引号逃逸
        if re.search(self.SQL_EVASION, payload):
            confidence += 0.2
            matches.append("SQL Evasion Pattern")
            
        # 3. REFLECT 数据：检查响应体是否返回了数据库错误 (高置信度确认)
        if re.search(self.SQL_ERRORS, response_body, re.IGNORECASE):
            confidence = 1.0  # 确认无误
            matches.append("DB Error Disclosure")
            
        return {
            "signature": "SQL_Injection_Attempt",
            "confidence": min(confidence, 1.0),
            "evidence": matches,
            "payload_sample": payload[:100]  # 截取前100字符作为样本
        }

class XSSPatternDetector:
    """
    XSS 攻击特征检测
    """
    XSS_PATTERNS = [
        r'<script[^>]*>',
        r'javascript:',
        r'onerror\s*=',
        r'onload\s*=',
        r'alert\s*\(',
        r'document\.cookie',
        r'<img[^>]*src\s*='
    ]
    
    def detect(self, payload: str, response_body: str) -> dict:
        confidence = 0.0
        matches = []
        
        for pattern in self.XSS_PATTERNS:
            if re.search(pattern, payload, re.IGNORECASE):
                confidence += 0.25
                matches.append(f"XSS Pattern: {pattern}")
                break
        
        # 检查响应中是否反射了payload（高置信度）
        if payload in response_body:
            confidence += 0.5
            matches.append("Payload Reflected in Response")
            
        return {
            "signature": "XSS_Attempt",
            "confidence": min(confidence, 1.0),
            "evidence": matches,
            "payload_sample": payload[:100]
        }

class PathTraversalDetector:
    """
    路径遍历攻击检测
    """
    TRAVERSAL_PATTERNS = [
        r'\.\./',
        r'\.\.\\',
        r'%2e%2e/',
        r'%2e%2e%5c',
        r'\.\.%2f',
        r'etc/passwd',
        r'boot\.ini',
        r'win\.ini'
    ]
    
    def detect(self, payload: str, response_body: str) -> dict:
        confidence = 0.0
        matches = []
        
        for pattern in self.TRAVERSAL_PATTERNS:
            if re.search(pattern, payload, re.IGNORECASE):
                confidence += 0.4
                matches.append(f"Traversal Pattern: {pattern}")
                break
                
        return {
            "signature": "Path_Traversal_Attempt",
            "confidence": min(confidence, 1.0),
            "evidence": matches,
            "payload_sample": payload[:100]
        }
```

### 4.5 REACT 引擎实现

```python
# agent/specialist/react_engine.py
from typing import Optional, List
from dataclasses import dataclass

@dataclass
class FocusArea:
    """REACT 引擎确定的分析焦点"""
    tool_name: str
    payload: str
    reason: str

class TrafficReactEngine:
    """
    流量分析的 REACT 推理引擎
    负责决定下一步检查哪个部分
    """
    
    def __init__(self):
        self.scanned_parts = set()
    
    def think(self, flow_sample: dict, existing_findings: List) -> Optional[FocusArea]:
        """
        推理下一步要分析的重点
        """
        # 1. 优先检查 URL 参数
        url_params = self._extract_url_params(flow_sample.get("url", ""))
        for param_name, param_value in url_params.items():
            key = f"param_{param_name}"
            if key not in self.scanned_parts:
                self.scanned_parts.add(key)
                tool = self._select_tool_for_param(param_value)
                if tool:
                    return FocusArea(
                        tool_name=tool,
                        payload=param_value,
                        reason=f"URL parameter '{param_name}' shows suspicious patterns"
                    )
        
        # 2. 检查 POST 请求体
        post_data = flow_sample.get("request_body", "")
        if post_data and "body" not in self.scanned_parts:
            self.scanned_parts.add("body")
            tool = self._select_tool_for_param(post_data)
            if tool:
                return FocusArea(
                    tool_name=tool,
                    payload=post_data,
                    reason="POST body contains executable or database patterns"
                )
        
        # 3. 检查请求头
        headers = flow_sample.get("headers", {})
        for header_name, header_value in headers.items():
            key = f"header_{header_name}"
            if key not in self.scanned_parts:
                self.scanned_parts.add(key)
                tool = self._select_tool_for_param(header_value)
                if tool:
                    return FocusArea(
                        tool_name=tool,
                        payload=header_value,
                        reason=f"Header '{header_name}' may be vulnerable"
                    )
        
        return None  # 所有部分都已扫描
    
    def _extract_url_params(self, url: str) -> dict:
        """提取 URL 参数"""
        from urllib.parse import urlparse, parse_qs
        parsed = urlparse(url)
        return {k: v[0] if v else '' for k, v in parse_qs(parsed.query).items()}
    
    def _select_tool_for_param(self, value: str) -> Optional[str]:
        """根据参数特征选择合适的检测工具"""
        import re
        
        # SQL注入特征
        if re.search(r'[\'";]|select|union|insert', value, re.IGNORECASE):
            return "sql_injection"
        
        # XSS特征
        if re.search(r'[<>]|script|javascript|onerror', value, re.IGNORECASE):
            return "xss"
        
        # 路径遍历特征
        if re.search(r'\.\./|\.\.\\|etc/passwd', value, re.IGNORECASE):
            return "path_traversal"
        
        return None
```

## 8. 总结

本文档定义了一个严格遵守多 Agent Runtime 架构的 **Passive Traffic Audit Agent**。通过引入 **REACT + REFLECT** 的推理机制，该模块能够在保持低误报率的同时，精准识别 HTTP 流量中的高风险线索。作为 `Specialist Agent`，它完全解耦于 API 层，仅通过 `Orchestrator` 进行调度，确保了后端系统的稳定性、可扩展性和可审计性。
